# KaggleWorks: My Machine Learning Learning Adventure

Welcome to my ML journey repository.

This workspace is a collection of practical notebook projects where I learned by doing: exploring real datasets, understanding patterns, cleaning messy data, building models, and evaluating results. Instead of only reading theory, I practiced full workflows repeatedly until concepts became natural.

This repo contains multiple EDA + ML notebooks across different domains:

## What I Learned (Core Skills)

### 1) EDA (Exploratory Data Analysis)
I learned how to:
- Inspect shape, columns, data types, and missing values
- Summarize data with descriptive statistics
- Detect outliers and skewed distributions
- Analyze relationships using correlation and grouped views
- Use visualizations to discover hidden patterns

### 2) Data Cleaning and Preparation
I practiced:
- Missing value handling strategies (drop, fill, median/mode, etc.)
- Removing duplicates and irrelevant columns
- Encoding categorical features (label encoding / one-hot)
- Feature scaling and normalization
- Train/validation/test splitting with correct methodology

### 3) Machine Learning Pipeline Thinking
A major outcome was understanding the full pipeline:
1. Problem understanding
2. Data loading and quality checks
3. EDA and hypothesis building
4. Preprocessing and feature engineering
5. Model training
6. Evaluation and comparison
7. Interpretation and improvement
8. Submission/reporting (for competition tasks)

This pipeline mindset is one of the most valuable things I gained.

### 4) Supervised Learning
Across projects, I practiced both classification and regression:
- Classification examples: Income, Diabetes, Car Evaluation, Titanic
- Regression examples: Housing and car price/sales tasks

I learned to evaluate with appropriate metrics depending on task type.

### 5) Unsupervised Learning
In `Unsupervised_Learning.ipynb`, I worked on clustering and segmentation ideas.

I learned:
- How unsupervised learning differs from supervised learning
- Cluster intuition and interpretation
- How to validate clustering quality with visual and metric-based checks

### 6) Model Evaluation and Improvement
I gained hands-on experience with:
- Accuracy, precision, recall, F1-score
- Confusion matrix analysis
- Regression metrics such as MAE/MSE/RMSE and R2
- Cross-validation mindset
- Detecting overfitting/underfitting and improving models

### 7) Kaggle Competition Workflow
Through the Titanic project in `Competition/`, I practiced:
- Separate train/test handling
- Feature preparation for real competition format
- Generating prediction submission files
- Working in a practical, results-driven loop

## Biggest Takeaways From This Adventure

1. Good EDA improves model quality more than random model switching.
2. Data preprocessing is as important as algorithm choice.
3. The same ML pipeline can be adapted to many domains.
4. Evaluation metrics must match the business/problem objective.
5. Iteration (analyze -> model -> evaluate -> improve) is the real learning engine.
